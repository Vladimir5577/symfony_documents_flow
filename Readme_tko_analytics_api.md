# TKO Analytics API (SPA)

Документация SPA-эндпоинта недельной сводки аналитики ТКО по всем полигонам.

## Базовый URL

```
https://<domain>/spa/api/analytics/tko
```

## Аутентификация

SPA-эндпоинты под префиксом `/spa/api/*` используют **JWT** (firewall `spa_api`), а не сессию.

**Получение токена:**
```
POST /spa/api/login_check
Content-Type: application/json

{ "login": "<login>", "password": "<password>" }
```
Ответ: `{ "token": "<jwt>", "refresh_token": "<refresh>" }`

**Заголовок авторизации для всех запросов:**
```
Authorization: Bearer <jwt_token>
```

**Требуется роль:** `ROLE_MANAGER` (атрибут `#[IsGranted('ROLE_MANAGER')]` на контроллере).

## Эндпоинт: недельная сводка по полигонам

```
GET /spa/api/analytics/tko/week?week=2026-05-18
```

Возвращает сводку за **одну неделю**: одна строка на каждый активный полигон + итог по всем.
Колонки — метрики ТКО. Числовые метрики суммируются за неделю (`agg: "sum"`), текстовые
отдаются как число дней недели с заполненным значением (`agg: "daysCount"`).

**Route name:** `spa_api_analytics_tko_view_summary`

> Не путать с `GET /spa/api/analytics/tko/summary` (route `spa_api_analytics_tko_summary`) —
> это другой эндпоинт: многонедельный календарь с `limit`/`offset` и древовидной разбивкой
> по полигонам. Здесь же — плоская сводка ровно за одну неделю.

### Query-параметры

| Параметр | Тип    | Обязательное | Описание                                                                 |
|----------|--------|--------------|--------------------------------------------------------------------------|
| `week`   | string | Нет          | Любая дата недели в формате `Y-m-d`. Нормализуется к понедельнику этой недели. При отсутствии/некорректном значении берётся текущая неделя. |

**Период:** неделя всегда Пн–Вс. `week=2026-05-18` и `week=2026-05-21` дадут одну и ту же
неделю (`start: 2026-05-18`, `end: 2026-05-24`).

### Ответ `200 OK`

```json
{
  "week": {
    "start": "2026-05-18",
    "end": "2026-05-24",
    "prev": "2026-05-11",
    "next": "2026-05-25"
  },
  "metrics": [
    { "key": "garbage_trucks_volume", "name": "Мусоровозы",        "unit": "м³",  "agg": "sum" },
    { "key": "garbage_trucks_weight", "name": "Вес ТКО мусоровозы", "unit": "т",   "agg": "sum" },
    { "key": "machinery_work",        "name": "Работа техники",     "unit": "дн.", "agg": "daysCount" }
  ],
  "rows": [
    {
      "polygonId": 1,
      "name": "Полигон №1",
      "values": {
        "garbage_trucks_volume": 1240,
        "garbage_trucks_weight": 318.5,
        "machinery_work": 5
      }
    },
    {
      "polygonId": 2,
      "name": "Полигон №2",
      "values": {
        "garbage_trucks_volume": 860,
        "garbage_trucks_weight": null,
        "machinery_work": 2
      }
    }
  ],
  "totals": {
    "garbage_trucks_volume": 2100,
    "garbage_trucks_weight": 318.5,
    "machinery_work": 7
  }
}
```

> В примере метрики и значения сокращены для краткости. Фактически в `metrics`, в каждом
> `rows[].values` и в `totals` присутствуют **все 13 метрик** (см. таблицу ниже) — для
> каждого ключа всегда есть поле, даже если значение пустое (`null`).

### Поля ответа

| Поле       | Тип      | Описание                                                                  |
|------------|----------|---------------------------------------------------------------------------|
| `week`     | object   | Границы периода и навигация (все даты — `Y-m-d`). См. «Модель Week».       |
| `metrics`  | object[] | Описание метрик (колонок) в фиксированном порядке. См. «Модель Metric».    |
| `rows`     | object[] | По одной строке на каждый активный полигон. См. «Модель Row».             |
| `totals`   | object   | Итог по всем полигонам: `{ <metricKey>: number\|null }` для каждой метрики. |

#### Модель Week (`week`)

| Поле    | Тип    | Описание                                                |
|---------|--------|----------------------------------------------------------|
| `start` | string | Понедельник выбранной недели (`Y-m-d`).                 |
| `end`   | string | Воскресенье выбранной недели (`Y-m-d`).                 |
| `prev`  | string | Понедельник предыдущей недели — для навигации «назад».  |
| `next`  | string | Понедельник следующей недели — для навигации «вперёд».  |

#### Модель Metric (`metrics[]`)

| Поле   | Тип    | Описание                                                                       |
|--------|--------|--------------------------------------------------------------------------------|
| `key`  | string | Ключ метрики. Совпадает с ключами в `values` и `totals`.                       |
| `name` | string | Название метрики для отображения.                                              |
| `unit` | string | Единица измерения: `м³`, `т`, `руб.`, `дн.`.                                   |
| `agg`  | string | Семантика колонки: `sum` — сумма за неделю; `daysCount` — число дней с отметкой. |

#### Модель Row (`rows[]`)

| Поле        | Тип    | Описание                                                                      |
|-------------|--------|-------------------------------------------------------------------------------|
| `polygonId` | int    | ID полигона (стабильный ключ для списков, drill-down).                        |
| `name`      | string | Название полигона.                                                            |
| `values`    | object | `{ <metricKey>: number\|null }` — значение по каждой метрике. Нет данных — `null`. |

**Типы значений:** все значения в `values` и `totals` — **числа** (`int` для целых, `float`
для дробных) либо `null`, если данных нет. `null` означает именно отсутствие данных, не ноль.

## TypeScript-тип

```ts
type Agg = "sum" | "daysCount";

interface TkoMetric {
  key: string;
  name: string;
  unit: string; // "м³" | "т" | "руб." | "дн."
  agg: Agg;
}

interface TkoSummaryRow {
  polygonId: number;
  name: string;
  values: Record<string, number | null>;
}

interface TkoSummaryResponse {
  week: { start: string; end: string; prev: string; next: string };
  metrics: TkoMetric[];
  rows: TkoSummaryRow[];
  totals: Record<string, number | null>;
}
```

## Ошибки

| Код | Описание                                                       |
|-----|----------------------------------------------------------------|
| 401 | Не авторизован (нет токена / невалидный или истёкший JWT).      |
| 403 | Нет роли `ROLE_MANAGER`.                                        |

`week` не валидируется как ошибка: некорректное значение молча трактуется как текущая неделя.

## Примечания для фронтенда

- Колонки таблицы строить из `metrics` (в полученном порядке); единица измерения — `unit`.
- Строки — `rows`, плюс зафиксированная снизу строка итога `totals`.
- Значение метрики в строке: `rows[i].values[metric.key]`. `null` → отрисовать пустую ячейку.
- Презентация — на фронте: формат чисел и подпись периода (например, `«18.05 — 24.05»` из
  `week.start`/`week.end`) бэк намеренно не отдаёт.
- Навигация по неделям: кнопки «назад/вперёд» подставляют `week.prev` / `week.next` в параметр `week`.
- `polygonId` использовать как ключ списка и для перехода на детальную неделю полигона.
