================================================================================
SPA API — Персональные папки и перемещения проектов
================================================================================

Эндпоинты для управления персональными папками и порядком проектов в боковом меню SPA-клиента. 
Все папки и сортировка привязаны к конкретному пользователю индивидуально.

Авторизация: JWT, ROLE_USER.


--------------------------------------------------------------------------------
СТРУКТУРА МЕНЮ (ПОЛУЧЕНИЕ ДАННЫХ)
--------------------------------------------------------------------------------

  Данные для построения дерева папок и проектов возвращаются в эндпоинте:
  GET /spa/api/me (см. dev_docks/spa_api/Readme_spa_auth.txt)

  В ответ добавлены новые поля:

  1. Поле projectFolders (массив папок текущего пользователя):
     "projectFolders": [
       { "id": 1, "name": "Важное", "position": 1.0 },
       { "id": 2, "name": "Архив", "position": 2.0 }
     ]
     Отсортировано по возрастанию position.

  2. Поля внутри каждого элемента массива projects:
     "folderId" — ID папки (int или null; null означает, что проект лежит в корне)
     "position" — сортировка проекта внутри его папки/корня (float)


--------------------------------------------------------------------------------
СОЗДАНИЕ ПАПКИ ПРОЕКТОВ
--------------------------------------------------------------------------------

  Method : POST
  URL    : /spa/api/project-folders

  Body:
  {
    "name": "Новая папка"
  }

  name — обязательно, непустая строка, максимум 255 символов.
  Позиция (position) выставляется автоматически (+1.0 к максимальной существующей).

--- Ответ (201) ----------------------------------------------------------------

{
  "id": 5,
  "name": "Новая папка",
  "position": 3.0
}


--------------------------------------------------------------------------------
РЕДАКТИРОВАНИЕ ПАПКИ (Переименование)
--------------------------------------------------------------------------------

  Method : PATCH
  URL    : /spa/api/project-folders/{id}

  Body:
  {
    "name": "Другое название"
  }

--- Ответ (200) ----------------------------------------------------------------

{
  "id": 5,
  "name": "Другое название",
  "position": 3.0
}


--------------------------------------------------------------------------------
УДАЛЕНИЕ ПАПКИ
--------------------------------------------------------------------------------

  Method : DELETE
  URL    : /spa/api/project-folders/{id}

  При удалении папки все проекты, лежащие в ней, автоматически перемещаются в корень
  (их folderId становится null).

--- Ответ (200) ----------------------------------------------------------------

{
  "success": true
}


--------------------------------------------------------------------------------
ПЕРЕМЕЩЕНИЕ И СОРТИРОВКА ПАПОК
--------------------------------------------------------------------------------

  Служит для перетаскивания (Drag & Drop) самих папок в боковом меню.

  Method : PATCH
  URL    : /spa/api/project-folders/{id}/move

  Body:
  {
    "position": 1.5
  }

  position — дробное число (float), новое место папки в списке.
  При накоплении погрешности на бэкенде автоматически сработает ребалансировка.

--- Ответ (200) ----------------------------------------------------------------

{
  "id": 5,
  "name": "Другое название",
  "position": 1.5
}


--------------------------------------------------------------------------------
ПЕРЕМЕЩЕНИЕ И СОРТИРОВКА ПРОЕКТОВ
--------------------------------------------------------------------------------

  Служит для перемещения проекта в папку или корень, а также для сортировки проектов.

  Method : PATCH
  URL    : /spa/api/projects/{id}/move

  Body:
  {
    "folderId": 2,
    "position": 1.5
  }

  folderId — ID папки назначения (int или null; null — перенос в корень)
  position — дробное число (float), новая позиция проекта внутри целевой папки/корня.

--- Ответ (200) ----------------------------------------------------------------

{
  "id": 3,
  "folderId": 2,
  "position": 1.5
}

--- Ошибки ---------------------------------------------------------------------

  400  { "error": "Некорректный JSON" } (если не передан position)
  403  { "error": "Доступ запрещен" } (попытка переместить проект чужого пользователя)
  404  { "error": "Проект не найден" } / { "error": "Папка не найдена" }
